\section{Spezifizierte Anforderungen}

\subsection{Funktionale Anforderungen}
Diese Anforderungen sollen vor allem für einen schnellen Überblick sorgen und die technischen Fragen klären.

\subsubsection{Zugrunde liegende Software}
\begin{description}
	 \item[Webserver:] Apache 2.0.xx mit PHP4 (für das Forum) und mod\_jk (als Verbindung zum Servlet-Container)
	\item [Datenbankserver:] MySQL5.0.x (wichitg vor allem für Views)
	\item [Servlet-Container:] Apache Tomcat 5.5 mit J2SE5.0
 \end{description}

\subsubsection{Bildergalerien}
\label{sec:bildergalerie}
\begin{itemize}
 \item Galerie/Untergalerie öffnen
 \item Bild betrachten
 \item Bild bewerten (AJAX)
 \item Direktlink für Bild erhalten
 \item Bilder sind nicht direkt herunterladbar
 \item Benutzer kann sich 10 Bilder zumailen lassen
	\begin{itemize}
		\item 10 Bilder in einen Korb verschieben
		\item Korb anzeigen lassen
		\item Captcha zur Überprüfung
		\item Emailadresse angeben und Bilder zumailen lassen
	\end{itemize}
 \item Top50 betrachten
 \item Admin kann
	\begin{itemize}
 		\item Galerie anlegen (Applet zum Upload von Dateien?)
		\begin{itemize}
			\item Thumbnails generieren
			\item Wasserzeichen einfügen
			\item Thumbnail für Übersicht auswählen
		\end{itemize}
		\item Galerie bearbeiten
			\begin{itemize}
			 \item Untergalerie hinzufügen
 			\item Datum, Fotograf und Titel angeben
 			\item Cavebot kann Thread zum Kommentieren anlegen
			\end{itemize}
		\item Galerie löschen
		\item Foto aus einer Galerie löschen
	\end{itemize}
\end{itemize}

\label{sec:gaestebuch}

\end{itemize}

\subsection{Nicht-funktionale Anforderungen}
Obwohl oder gerade weil es sich um eine private Seite handelt müssen die Gesetzte eingehalten werden. Vor allem das DatenschutzG sind dabei zu berücksichtigen.

Auch soll Incave.de barrierfrei sein. Einerseits aus moralischen Gründen, andererseits ist davon auszugehen, dass immer mehr mobile Geräte auch das Internet nutzen können. Dann ist eine Verwendung von Incave direkt von unterwegs (z.B. direkt auf einer Party) problemlos möglich.

Das bedeutet, das die meisten Funktionalitäten mit reinem XHTML und CSS2 realisiert werden. JavaScript sollte nur dann eingsetzt werden, wenn es sich nicht anders realisieren lässt bzw wenn das nicht-funktionieren von JavaScript nur optishe Einbußen mit sich bringt. Auch ist daran zu denken, dass kein Browser 100\% XHTML- und CSS2-kompatibel ist.

\subsection{Externe Schnittstellen}
Das Forum wird vom sog. CaveBot dazu genutzt, um für bestimmte Dinge (Bildergallerie, Kalendereintrag) einen Thread zu eröffnen. Dies wird durch direkte Modifikation der Tabellen vom WBB-Forum erreicht. Abzuklären ist, ob es irgendwelche Wrapper-Funktionen, möglichst in Java bzw. JSP dazu gibt.

\subsection{Qualitätsanforderungen}
Die Besucherzahlen auf der Seite schwanken sehr. An normalen Tagen sind nur die Stammbesucher vertrten, die hauptsächlich das Forum nutzen. An Tagen nach Parties, vor allen am Wochenende, kommen aber durchaus bis zu 1000 Leute pro Tag die Seite. Vor allem von diesem "`Laufkundschaft"' werden lange Wartezeiten, Fehler oder sogar Downtimes nicht akzeptiert. Ein reibungsloser Systemebetrieb ist extrem wichtig!

\subsection{Sonstige Anforderungen}
Die zukünfitge Wartbarkeit und Erweiterbarkeit von Incave.de darf nicht be- oder verhindert werden. Daher ist es zwingend erforderlich, dass alles ausführlichst dokumentiert ist.
